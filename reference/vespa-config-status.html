This is the reference documentation for the command-line tool vespa-config-status.

The command line tool is called vespa-config-status. It can run on any machine in a Vespa cluster and will output a list of all the running services which is running with an outdated application package. The tool can be invoked without any arguments, or you can pass it optional arguments. The arguments are given below.

-v
Show all servies, even if they are up to date.
-c host[:port]
Get Vespa cluster configuration from the config server specified by host and port.

An example session with this tool may look like the following: 

$ vespa-config-status
Subscribing to model config with config id admin/model
search/qrsclusters/default/qrserver.0:container has the latest generation 1
