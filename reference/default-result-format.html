This document describes the default result format returned by Vespa JDisc Container nodes. This result format is returned when no presentation.format parameter is given in the query, see the search API for details.

The default result format is JSON, the XML format is deprecated.

Default JSON Format

The JSON result format is activated by setting presentation.format to JsonRenderer or json.

Results are rendered as a JSON object with two fields, the mandatory root field which contains the tree of returned data, and the optional trace field for storing meta data about query execution. All field names here are literal strings, the node root is the map key “root” in the return JSON object, in other words, only strings are used as map keys.

  • root mandatory, map of string to object
    The root of the tree of returned data.
    • children optional, array of objects
      Array of JSON objects with exactly the same structure as root.
    • coverage optional, map of string to string and number
      Metadata about how much of the total corpus has been scanned to return the given documents.
      • coverage mandatory, integer
        Percentage of total corpus scanned.
      • documents mandatory, long
        The number of documents scanned.
      • full mandatory, boolean
        Whether the full corpus was scanned.
      • nodes mandatory, integer
        The number of search instances used executing the query.
      • results mandatory, integer
        The number of results merged creating the final rendered result.
      • resultsFull mandatory, integer
        The number of full result sets merged.
    • errors optional, array of objects
      Array of error messages with the fields given below.
      • code mandatory, integer
        Numeric identifier used by the container application.
      • message optional, string
        Full description.
      • source optional, string
        Which data provider logged the error condition.
      • stackTrace optional, string
        Stack trace if an exception was involved.
      • summary mandatory, string
        Short description.
      • transient optional, boolean
        Whether the system is expected to recover from the faulty state on its own. If the flag is not present, this may or may not be the case, or the flag is not applicable.
    • fields optional, map of string to object
      The named document fields, this where fields of search documents go.
    • id optional, string
      String or URI identifying the hit, document or other data type.
    • label optional, string
      The label of a grouping list.
    • limits optional, object
      Used in grouping, the limits of a bucket in histogram style data.
      • from optional, string
        Lower bound of a bucket group.
      • to optional, string
        Upper bound of a bucket group.
    • relevance mandatory, double
      Double valure representing the rank score.
    • source optional, string
      Which data provider created this node.
    • types optional, array of string
      Meta data about what kind of document or other kind of node in the result set this object is.
    • value optional, string
      Used in grouping for value groups, the argument for the grouping data which is in the fields.
  • trace optional, map of string to object
    Meta data about query execution.
    • children optional, array of object
      Array of maps with exactly the same structure as trace itself.
    • timestamp optional, long
      Number of milliseconds since the start of query execution this node was added to the trace.
    • message optional, string
      Descriptive text regarding this step of query execution.

XML Format

This XML format should be regarded as deprecated in favor of the JSON format.

The elements of the format are described below. Sub elements will be rendered in the order listed here. The format is open, all parsers must expect that all tags may contain attributes and child tags not mentioned here.

<result>

Contained in: —
Value: —
The root element of a default rendered result.

Attributes

  • total-hit-count

    Value:Integer
    Present:Always
    The estimated total number of hits matching this query

Sub Elements

Element Present
<error> One if there is a single error, otherwise none
<errordetails> One if there are multiple errors, otherwise none
<group> This represents hierarchies of hits.
<hit> One per hit (excluding hits returned in other ways)

<error>

Contained in: <errordetails>, <result>
Value: A human readable error message. The message will be generic when this is the top level error, and more detailed when this is inside <errordetails>
Describes an error occurring during the search. Error messages may appear instead of, or in addition to normal results.

Attributes

  • code

    Value:A positive integer
    Present:Always
    The error code. Error codes will remain stable.

  • source

    Value:
    Present:When a source is assigned for this error
    The name of the search source where this error occured

  • error

    Value:
    Present:Always if this error is inside <errordetails>, never when this is the top level error
    The error message.

Sub Elements

None.

<errordetails>

Contained in: <result>
Value: —
Container element for <error> elements.

Attributes

None.

Sub Elements

Element Present
<error> One per occuring error

<group>

Contained in: <result>, <group>
Value: A tree of groups and hits where the hit instances are leaves.
Contains hierarchies of <hit> instances organized in groups.

Attributes

NameDescriptionPresent
relevancy A number between 0 and 1.0 (inclusive). 1 means perfectly relevant 0 means not relevant at all. Always

Sub Elements

Element Present
<hit> Normal documents or grouping data.
<group> New groups of hits.

<hit>

Represents a normal query match.

Attributes:

NameDescriptionPresent
relevancy A number between 0 and 1000 (inclusive). 1000 means perfectly relevant 0 means not relevant at all. Always
source The name of the source which created this hit Always
type A space-separated list of the application-specific types assigned to this hit. Normal hits representing the summary of an indexed result has (at least) the type summary If the hit has any types set

Subtags:

NameDescriptionPresent
field A property of this hit, e.g. summary fields One per property of the hit

<field>

A name-value property of a hit. This includes normal summary fields and any other application-specific property added to a hit.

Attributes:

NameDescriptionPresent
NameThe name of the propertyAlways

Content:

The value of the field

Examples

An empty (no hits) result

<?xml version="1.0" encoding="utf-8" ?>
<result total-hit-count="0">
</result>

A result containing an error and hits.

<?xml version="1.0" encoding="utf-8" ?>
<result total-hit-count="2">
  <error code="55">Some error message</error>
  <hit relevancy="75" source="test" type="summary">
    <field name="uri">http://def</field>
    <field name="title">Moonlight in the sky</field>
  </hit>
  <hit relevancy="73" source="test" type="summary test other">
    <field name="uri">http://a.b/c</field>
    <field name="category">test/stufftsome/other</field>
    <field name="title">Kanoos and sunshine</field>
  </hit>
</result>

A result containing error information, hierarchal hits, grouping information, tracing...

<?xml version="1.0" encoding="utf-8" ?>
<result total-hit-count="0" coverage-docs="500" coverage-nodes="1" coverage-full="true" coverage="100" results-full="1" results="1" querytime="9223370674710200.000" summaryfetchtime="-9223370674710200.000" searchtime="0.078">
  <error code="18">Internal server error.</error>
  <meta type="context">
    <p>No query profile is used</p>
    <p>Resolved properties:&lt;br/&gt;
reportCoverage=true (value from request)&lt;br/&gt;

tracelevel=5 (value from request)&lt;br/&gt;
</p>
    <p>Detected language: ENGLISH</p>
    <p>Language ENGLISH determined by the characters in the terms.</p>
  </meta>
  <meta type="meta:Query feedback">
    <p>feedback</p>
  </meta>
  <errordetails>
    <error error="Internal server error." code="18">boom</error>
  </errordetails>
  <group relevancy="1.0">
    <hit type="summary" relevancy="0.9">
      <field name="relevancy">0.9</field>
      <field name="uri">http://localhost/1</field>
      <field name="c">d</field>
    </hit>
  </group>
  <hit type="grouphit" relevancy="1.0">
    <id>type grouphit</id>
  </hit>
  <hit type="summary" relevancy="0.95">
    <field name="relevancy">0.95</field>
    <field name="uri">http://localhost/</field>
    <field name="b">foo</field>
  </hit>
</result>
