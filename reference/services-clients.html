<abortondocumenterror>

Contained in: <feederoptions> or <spooler>
Value: xsd:boolean
Default Value: true
Quantity: One

Controls whether or not to abort the entire feed if a document-related error occurs, ie. if a document contains an unknown field. Setting this field to true will abort the feed on such errors, while setting it to false will cause Vespa to simply skip to the next document in the feed.

Note that malformed XML in the input will abort the feed regardless of this setting.

Attributes

None.

Sub Elements

None.

Examples

<feederoptions>
  <abortondocumenterror>false</abortondocumenterror>
  <timeout>60</timeout>
</feederoptions>

<maxpendingbytes>

Contained in: <feederoptions> or <spooler>
Value: Any positive Integer
Default Value: —
Required: Yes
Since: No
Deprecated:
Quantity:

The maximum number of pending bytes. If <maxpendingdocs> is 0 and this is set to 0, this defaults to 100 MB. If <maxpendingdocs> is more than 0, and this is set to 0, the send window is only limited by number of messages sent, not the memory footprint.

Attributes

None.

Sub Elements

None.

Examples

<feederoptions>
  <maxpendingbytes>1048576</maxpendingdocs>
<feederoptions>

<maxpendingdocs>

Contained in: <feederoptions> or <spooler>
Value: Any positive Integer
Default Value: Automatically adjusted
Quantity: One

The maximum number of pending documents the client can have. By default, the client will dynamically adjust the window size based on the latency of the performed operations. If the parameter is set, dynamic window sizing will be turned off in favor of the configured value.

Attributes

None.

Sub Elements

None.

Examples

<feederoptions>
  <maxpendingdocs>1000</maxpendingdocs>
<feederoptions>

<parameter>

Contained in: <parser>
Value: —
Default Value: —
Quantity: Zero or more

FileParser classes may be configured using the parameters set using this element. The specified parameters may be accessed using the getParameters() method. Each parameter consists of a key-value pair, described in the below list of attributes.

Attributes

  • key

    Value:A non-zero length String
    Default Value:—
    Required:Yes

    The name of the parameter.

  • value

    Value:A non-zero length String
    Default Value:—
    Required:Yes

    The value of the parameter.

Sub Elements

None.

Examples

<parser type="com.yahoo.mail.vespa.MigrationParser">
    <parameter key="route" value="docproc storage" />
</parser>

<parser>

Contained in: <parsers>
Value: —
Default Value: —
Required: Yes
Quantity: One or more

Used for configuring a Spooler to use a specific parser. Parsers can either be built-in (found in com.yahoo.vespaspooler.*) or be part of a JAR file located in $VESPA_HOME/libexec/vespa/plugins/spooler.

Attributes

  • type

    Value:String
    Default Value:—
    Required:Yes

    Sets the parser type. This must be the full class name of the FileParser class.

Sub Elements

Element Required Quantity
<parameter> Zero or more

Examples

  <parser type="com.yahoo.mail.vespa.MigrationParser">
    <parameter key="route" value="docproc storage" />
  </parser>
</parsers>

<spooler>

Contained in: <spoolers>
Value: —
Default Value: —
Required: Yes
Quantity: One or more

Configure a VespaSpooler.

Attributes

  • directory

    Value:String
    Default Value:—

    The directory used by this spooler. See the VespaSpooler Usage documentation for more in-depth information.

  • id

    Value:String
    Default Value:—

    Enter description

  • keepsuccess

    Value:Boolean
    Default Value:—

    Toggle whether to keep all successfully parsed files in the successes/ subdirectory or not.

  • maxfailuresize

    Value:Integer
    Default Value:Unlimited

    Configure the maximum number of bytes that may be stored in the failure/ directory. If this is exceeded, failures will only be logged. By default all failures are kept.

  • maxfatalfailuresize

    Value:Integer
    Default Value:Unlimited(?)

    Configure the maximum number of bytes that may be stored for failures of type FATAL. This allows you to only save a certain amount of failure (by setting the maxfailuresize above) bu still keep all failures that are fatal.

  • maxretries

    Value:Integer
    Default Value:—

    The number of retries in case of transient errors when feeding to the Vespa backend.

  • threads

    Value:Integer
    Default Value:—

    Configure the number of threads to use.

  • baseport

  • hostalias

  • jvmargs
