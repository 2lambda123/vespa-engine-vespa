Vespa search has by default limitations on how many hits you can request (maxHits) and how deep in the result set you can paginate (maxOffset).

Configure maxHits and maxOffset

Add a query profile named default saved as default.xml in your search/query-profiles/ directory of your application package with the following contents 
<query-profile id="default">
  <field name="maxHits">10</field>
  <field name="maxOffset">1000</field>
</query-profile>

re-deploy and enjoy the new limits. Note that increasing the number of hits and offset allowed will increase cpu and network usage.
