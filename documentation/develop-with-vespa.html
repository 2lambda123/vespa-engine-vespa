Developing with Vespa means writing Java plugin code for document and query processors - refer to the JDisc Container for details. Exploring the sample applications is a good start.

Try the tutorial to learn more about using different Vespa features. Vespa APIs is useful to understand how to interface with Vespa

Troubleshooting

Cannot find *Config classes

Several projects depend on generated configuration classes. These are created by the generate-sources Maven target. The easiest way to avoid the problem is building the project with Maven from the shell, then importing in Eclipse/IntelliJ. If you already have imported the project and want to get rid of the errors, first run the generate-sources target either from the IDE or the shell, then mark the pertinent target/generated-sources/vespa-configgen-plugin directories as source directories.
