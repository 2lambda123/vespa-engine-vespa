This guide will give you a single–node Vespa installation.

Requirements

  • A machine (real or virtual) running CentOS Linux node with at least 2GB RAM

  3. Start the Vespa config server: Start the Vespa config server, which needs to be up and running before deploying your application (wait five seconds after start, then proceed to next step):

    service vespa-configserver start

  4. Deploy an application: An example application is found at github.com/yahoo/vespa, deploy it:

    $ git clone git@github.com:yahoo/vespa.git
$ cd vespa/sample-apps/basic-search
$ deploy prepare src/main/application && deploy activate

  5. Start Vespa:

    service vespa start

  6. Create the index: Feed sample documents: 

    $ java -jar $VESPA_HOME/lib/jars/vespa-http-client-jar-with-dependencies.jar --file music-data-feed.json --host localhost

  7. Query the index:

    curl http://localhost:8080/search/?query=michael

Next Steps

Congratulations - Vespa is successfully installed and tested! Head right over to the Vespa Tutorial to learn more. Refer to indexing modes for streaming search applications.

More on documents and feeding

In step 6 above, the index was created, using the high-performance feed client. Read more in Vespa APIs. Try out a few commands from the Document Operation API:

  1. Dump documents:

    $ curl http://localhost:4080/document/v1/music/music/docid/

  2. Get a document:

    $ curl http://localhost:4080/document/v1/music/music/docid/Eminem-Recovery

  3. Delete a document:

    $ curl -X DELETE http://localhost:4080/document/v1/music/music/docid/Eminem-Recovery

  4. Put a new document:

    $ curl -X POST --data-binary @music-data-2.json http://localhost:4080/document/v1/music/music/docid/Eminem-So-Bad
