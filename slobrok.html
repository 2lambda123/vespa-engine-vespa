This document describes the Service Location Broker (Slobrok) service in Vespa.

Slobrok is an acronym for Service Location Broker, and it is a name service used in Vespa. The service listens on a specific port. Use the Vespa Model Inspector to find out what port number the Slobrok service has.

Clients (like the Document API), will try to do lookups toward any of the nodes the service location broker is running on. The Slobrok process looks like this: ps ax|grep slobrok: 

93906  ??  SJ     2:31.52 $VESPA_HOME/bin/slobrok -p 19100 -c slobrok.0

The Slobrok is running on the administration node as well as one or two other random nodes for redundancy. As the service requires minimal system resources, it should not disturb other services running on the same node.
